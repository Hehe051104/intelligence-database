  ---
  项目概览

  这是一个部署在 Cloudflare Workers 上的情报站网关服务，中文名叫"情报站"。它是一个云原生的智能情报聚合平台，主要功能是：
  1. 接收外部提交的情报数据
  2. 存储到 D1 数据库（结构化数据）
  3. 同时向 Vectorize 索引（向量数据库）插入向量表示
  4. 支持通过自然语言搜索情报

  ---
  通道 A: GET 请求（前端搜索接口）

  路径参数：
  - ?q=<搜索关键词> - 自然语言搜索
  - ?interest=<兴趣分类> - 按分类过滤（trending/某兴趣ID）

  工作流程：

  1. 无搜索词时 - 获取最新数据：
  SELECT * FROM intel_records
  WHERE interest_topic = ?  -- 如果有 interest 参数
  ORDER BY id DESC LIMIT 20

  2. 有搜索词时 - 语义搜索流程：
  用户输入 "AI 代理技术"
       │
       ▼
  ┌─────────────────┐
  │ AI Embedding 模型│ → 转为向量: [0.012, -0.456, ...]
  │ @cf/baai/bge-base-en-v1.5
  └─────────────────┘
       │
       ▼
  ┌─────────────────┐
  │ Vectorize 查询   │ → 相似度匹配 top 12 条
  │ semantic search  │
  └─────────────────┘
       │
       ▼
  ┌─────────────────┐
  │ 从 D1 获取完整   │
  │ 数据             │
  └─────────────────┘
       │
       ▼
  按匹配度排序返回

  关键细节：
  - 使用 bge-base-en-v1.5 模型（百度文心基座英文版，本地部署）
  - 搜索结果先按向量相似度匹配，再按匹配度排序
  - 支持分类过滤，与向量搜索结果交集

  ---
  通道 B: POST 请求（情报提交接口）

  认证方式：
  EXPECTED_TOKEN = "SPEC-2026-SuperSecretKey"
  Authorization: Bearer SPEC-2026-SuperSecretKey

  请求体结构：
  {
      title: string,           // 标题（必填）
      summary: string,         // 摘要（必填）
      url: string,             // 链接
      source_type: string,     // 来源类型
      tags: string,            // 标签
      importance_score: number,// 重要性分数
      tech_difficulty: number, // 技术难度
      social_value: number,    // 社会价值
      interest_id: string      // 兴趣分类 ID（可选，默认 trending）
  }

  双写逻辑（关键！）：

  // 步骤1: 生成向量嵌入
  embeddingResponse = await AI.run("@cf/baai/bge-base-en-v1.5", {
      text: [summary]  // 只用摘要生成向量
  })
  vector = embeddingResponse.data[0]  // 768维向量

  // 步骤2: D1 插入记录
  await DB.prepare(`
      INSERT INTO intel_records (
          id, title, summary, url, source_type, tags,
          importance_score, tech_difficulty, social_value, interest_topic
      ) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
  `).bind(id, title, summary, ...).run()

  // 步骤3: Vectorize 插入向量
  await VECTOR_INDEX.upsert([{
      id: id.toString(),
      values: vector,              // 768维向量
      metadata: {                   // 存储文本元数据
          title, tags, source_type,
          interest_topic: finalTopic
      }
  }])

  // 步骤4: 并行执行，然后等待完成
  await Promise.all([dbPromise, vectorizePromise])

  为什么双写？
  - D1 存储结构化数据（便于 SQL 查询、过滤、统计分析）
  - Vectorize 存储向量（便于语义搜索、推荐系统）

  核心技术点

  1. 为什么用 Cloudflare Workers？

  ┌──────────┬──────────────────────────────────────┐
  │   优势   │                 说明                 │
  ├──────────┼──────────────────────────────────────┤
  │ 边缘部署 │ 全球 300+ 节点，延迟 <50ms           │
  ├──────────┼──────────────────────────────────────┤
  │ 无服务器 │ 按请求付费，无需管理服务器           │
  ├──────────┼──────────────────────────────────────┤
  │ 内置 AI  │ Workers AI 提供 embedding 和生成模型 │
  ├──────────┼──────────────────────────────────────┤
  │ CDN 加速 │ 静态资源自动缓存                     │
  └──────────┴──────────────────────────────────────┘

  2. 为什么用 Vectorize？

  ┌──────────┬─────────────────────────────────────────┬──────────────────────┐
  │   对比   │                Vectorize                │    传统关键词搜索    │
  ├──────────┼─────────────────────────────────────────┼──────────────────────┤
  │ 查询方式 │ 语义相似度                              │ 关键词匹配           │
  ├──────────┼─────────────────────────────────────────┼──────────────────────┤
  │ 查询示例 │ "AI 技术趋势" → 自动匹配 "机器学习进展" │ 只能匹配 "AI" 关键词 │
  ├──────────┼─────────────────────────────────────────┼──────────────────────┤
  │ 性能     │ 亚毫秒级，分布式                        │ 取决于数据库         │
  └──────────┴─────────────────────────────────────────┴──────────────────────┘

  3. 为什么用 D1？

  - Cloudflare 自家的关系型数据库
  - 适合 SQL 查询、事务、外键
  - 与 Worker 无缝集成

  ---
  总结

  这是一个现代化的语义搜索情报系统，核心创新点：

  1. 双存储架构 - 结构化数据 + 向量搜索，兼顾准确性和灵活性
  2. 边缘 AI - 不需要自建 embedding 模型，直接用 Cloudflare AI
  3. 自动化清理 - 7 天自动过期，类似"情报时效性"的概念
  4. 实时推荐 - 支持自然语言搜索，而非关键词
